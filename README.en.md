Each keyword card contains **4 blocks**:
- 📖 **Pro definition** — 150-200 characters, three-part structure (definition + 2025-2026 data + industry impact)
- 🔍 **Dig deeper** — one-click jump to Wikipedia / Google
- 📈 **Industry data** — 35-70 characters with specific numbers and company names
- 💬 **Plain language** — 25-50 characters, everyday analogy

Each company card contains: **Eco-niche + Moat (Pro/Plain) + Lifecycle + Revenue structure (stacked bar)**

## 🚀 Quick Start

### Option A: Use as a WorkBuddy Skill (recommended)

1. Install locally:
```bash
git clone https://github.com/znzjLuo1008-netizen/industry-lens-skill.git \
  ~/.workbuddy/skills/industry-lens
```

2. In WorkBuddy chat, simply say:
```
Generate an industry encyclopedia for "Brain-Computer Interface"
IndustryLens for semiconductors
```

The skill auto-triggers and produces an HTML file within 3 minutes.

### Option B: Command line

```bash
# 1. Set your DeepSeek API key
export DS_KEY="sk-your-key"

# 2. Generate
python3 references/generate.py "Autonomous Driving"
# → Outputs to ./Autonomous Driving-百科.html

# Specify output path
python3 references/generate.py "Biotech" /tmp/bio.html

# Read from existing preloaded database (instant, no API)
python3 references/generate.py "Autonomous Driving" \
  --preloaded /path/to/preloaded_data.js
```

## 🎨 Design Principles

### Visual style
- **Minimalist light**: white surface, 0.5px hairline dividers, 16px radius
- **Main palette**: dark green `#166534` + teal `#0F6E56` + link blue `#185FA5`
- **Avoid**: colored gradient bars, multicolor capsule labels, emoji labels, heavy borders
- **Only the "plain language" block keeps a light-green tint** (as emotional anchor)

### Interaction
- **Auto number highlight**: percentages, autonomy levels get hyperlink styling (blue tint + underline)
- **Wiki / Google buttons**: dark-green text on light-green background, hover becomes dark-green filled
- **Card hover**: lift 3px + deepen shadow
- **IntersectionObserver scroll reveal**

### Copy guardrails
- **Research-report lens**: eco-niche / moat / lifecycle (no investment language)
- **Data must contain 2025-2026 concrete numbers + company names**, zero fabrication
- **Plain language**: needs analogy and wit, no short phrases

## ⚙️ Tech Stack

- **Pure vanilla HTML + inline CSS + JS** (no build, open with a double-click)
- **DeepSeek Chat API** for generation (any OpenAI-compatible API works)
- **Python 3** for the orchestration script
- **No database dependency** (all data inlined in HTML)

### API call tips
1. Use `curl -k` to skip SSL verification (enterprise networks)
2. Write payload to a temp file with `--data @file.json` to avoid shell escaping
3. 100 words need to be split into 2 calls (1-50 + 51-100) to avoid token truncation
4. Companies are a separate call (different schema)
5. Quality self-check: pro ≥130 chars, ref coverage 100%, companies have ≥3 revenue lines

## 🛠 Customization

### Visual tweaks
Edit the `<style>` block in `references/html_template.html`. Keep:
- Main colors `#0F6E56` / `#166534` unchanged (unless you want a different palette)
- Section separators: `border-top: 0.5px` rather than color blocks
- Class names like `.kw-num-hl` unchanged (JS depends on them)

### Change keyword count
In `generate.py`, `generate_keywords()` defaults to 2 batches × 50. Switch to 1 batch → 50 words, or change the prompt to "rank 1-30" → 30 words.

### Switch LLM
Edit `API_URL` and `body['model']` in `api_call.py`. Any OpenAI-API-compatible model works (Kimi / ChatGPT / Qwen / GLM / etc.).

## 💡 Known limits

- Data is bounded by LLM training cutoff around 2025, newer numbers may be stale
- Faces / person images are **not generated** in cards (only company logos via Clearbit)
- A full 100-word single call exceeds token limits, split required
- Some company logos may be missing from Clearbit → auto-fallback to initials
